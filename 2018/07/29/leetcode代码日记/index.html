1.two sum

万事开头难，第一题，两数求和，二分法。首先将数组进行排序，第一个索引的数加上最后一个索引的数，如果大于target，最后一个索引减1，如果小于target，索引i+1。m和n是找到对应解的原数组位置，如果m和n一样大说明数组类似于[3,3],index函数只会找到第一个等于目标值的索引，所以要把起始位置从m+1开始找。

64.minimum path sum

该题思路：从左上角到右下角找寻最短路径，首先初始化第一行，第一列，新建一个数组用于保存结果，第一行第一列到值等于左边值加本身或者上面值加本身，数组中间的值等于左边和上面的小的值。

771.Jewels and Stones

一道简单题，双重遍历时间复杂度O(n2)，或者使用hash，记录值时间复杂度O(n)

33.Search in Rotated Sorted Array

该题临界值是关键，旋转数组，不知道临界值在什么位置，但是临界值左边是有序的，临界值右边也是有序的。该题采用二分的思想，如果中间值小于最高位，说明右边是有序的，临界值在左边，如果target大于middle，low=mid+1，如果target小于middle，high=mid-1，同理可得到临界值在右边的情况。
